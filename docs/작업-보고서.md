## 2. 기존 구조 분석

기존 프로젝트는 401줄 단일 `index.html`과 문서 2개로 구성되어 있었습니다.

- `pipeline("zero-shot-image-classification")`로 통합 CLIP 그래프 사용
- 파일 마지막의 `loadModel()`이 페이지 접속 즉시 실행
- q8 실패 시 fp32 통합 모델로 폴백해 다운로드가 더 커질 가능성 존재
- 이미지 전처리, 텍스트 토큰화, 정규화, softmax는 고수준 파이프라인 내부 처리
- 캐시는 Transformers.js 기본 Cache Storage에 의존하나 상태 확인/삭제 UI 없음
- 추론이 메인 모듈에서 순차 실행
- 사진은 Object URL로만 처리되어 외부 전송 코드는 없음
- ZIP은 버튼 클릭 시 JSZip을 동적 import
- 초기화는 사진 Object URL과 결과만 삭제

## 3. 주요 구조 변경

1. 접속 시 `loadModel()` 호출을 제거했습니다.
2. `CLIPVisionModelWithProjection`과 `CLIPTextModelWithProjection`을 독립 로더로 분리했습니다.
3. 동일 모델 로딩은 같은 Promise를 재사용해 중복 다운로드를 막았습니다.
4. 기본 자동분류는 이미지 모델과 정적 카테고리 임베딩만 사용합니다.
5. 자유 검색은 사용자가 추가 다운로드에 동의한 뒤 텍스트 모델을 준비합니다.
6. 모델 로딩·이미지 전처리·추론·정규화·유사도 계산을 Web Worker로 옮겼습니다.
7. 이미지 임베딩은 Worker 메모리의 `Float32Array`로만 보관합니다.
8. Cache Storage를 검사해 이미지/텍스트 모델 설치 상태를 각각 표시합니다.
9. 모델 캐시 삭제와 사진 초기화를 분리했습니다.
10. q4/WebGPU/MobileCLIP은 비활성 실험 설정으로만 남겼습니다.

## 4. 다운로드 용량 전후 비교

공식 Hugging Face 파일의 `Content-Length`를 직접 측정했습니다.

| 항목 | bytes | decimal MB |
|---|---:|---:|
| 기존 통합 q8 | 153,695,702 | 153.7 |
| 이미지 q8 | 89,117,001 | 89.1 |
| 텍스트 q8 | 64,504,507 | 64.5 |

- 홈페이지 접속 모델 다운로드: 153.7MB → 0MB
- 기본 자동분류 모델 다운로드: 153.7MB → 89.1MB
- 감소량: 64,578,701 bytes, 약 42.0%
- 자유 검색 첫 사용 추가: 64.5MB
- 카테고리 임베딩 JSON: 114,312 bytes
- ONNX Runtime WASM 빌드 자산: 21,596,019 bytes, gzip 약 5.1MB

## 5. 기본 자동분류 다운로드 크기

`vision_model_quantized.onnx` 89,117,001 bytes입니다. 작은 config/processor 파일과 WASM 런타임 자산은 별도입니다.

## 6. 자유 검색 추가 다운로드 크기

## 8. 정확도/기능 테스트 결과

공개 테스트 이미지 2장을 프로젝트 밖 임시 폴더에 내려받아 사용했습니다.

- `cats.jpg`
- `football-match.jpg`

결과:

- 기본 자동분류: 고양이 → `고양이`, 축구 → `미분류`
- 자유 검색 `cat, football match`: 각각 올바른 검색 그룹으로 분리
- ZIP 생성: `cat.zip` 다운로드 확인
- 다중 파일 업로드, 임계값, 결과 그룹, 미분류, 초기화: 정상
- 유사도/정규화 단위 테스트: 4/4 통과

이 결과는 기능 스모크 테스트이며 정식 정확도 지표로 간주하지 않습니다.

## 10. 남은 문제와 권장 작업

1. 100~200장, 18개 이상 카테고리의 정식 정확도 벤치마크가 필요합니다.
2. Edge 전체 모델 다운로드/추론, Firefox, Safari 실기기 검증이 남았습니다.
3. 다운로드 진행 중 강제 취소는 현재 Transformers.js 로더 제약으로 제공하지 않고 오류 후 재시도를 제공합니다.
4. 기본 분류에서 축구 사진이 `여러 사람`이 아니라 미분류가 된 사례처럼 카테고리 프롬프트와 임계값을 실제 사진 세트로 보정해야 합니다.
5. q4/WebGPU/MobileCLIP은 별도 브랜치에서 정확도·메모리·브라우저 호환성 검증 후 판단해야 합니다.

## 11. 독립 점수 및 복수 라벨 추가

- 기존: 선택된 모든 카테고리가 하나의 softmax를 공유하고 1위 한 개만 채택
- 변경: 각 카테고리를 해당 카테고리의 부정 프롬프트와 독립 비교
- 결과: 한 사진이 여러 기준을 넘으면 여러 카테고리에 동시에 포함

통합된 기본 카테고리는 11개입니다.

1. 사람 — 여러 사람, 셀카, 아기/어린이 포함
2. 동물 — 고양이, 강아지 포함
3. 음식
4. 문서
5. 영수증
6. 풍경 — 바다, 산 포함
7. 건물
8. 도시
9. 실내
10. 야간
11. 스크린샷

차량 카테고리는 삭제했습니다.

사람+동물 합성 사진의 실제 Chrome 추론에서 사람 81%, 동물 83%가 측정되어 두 그룹에 동시에 포함되는 것을 확인했습니다. 독립 점수 전환 직후 관찰된 과검출을 줄이기 위해 카테고리별 보정 최소 점수 0.60~0.70을 추가했습니다.
